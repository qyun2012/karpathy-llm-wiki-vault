# Patient as Sensor（患者即传感器范式）

## 核心定义

将患者从**被动接收者**转变为**主动数据生产者**，融合日常 PRO、可穿戴设备数据、周度粪便标志物与定期深层 Omics 的**多源数据持续集成架构**，实现：

1. **早期 MRD 预警**：亚临床炎症升高在症状出现前被识别
2. **个性化干预**：基于患者特异的生物响应模式
3. **治疗遵从性监测**：检测患者是否按处方用药
4. **生活质量实时反馈**：PRO 数据驱动医生决策

## 四层数据采集架构

```
┌─────────────────────────────────────────────────┐
│ 层 1：日常 PRO 数据（每日）                      │
│ • 症状评分（腹痛、腹泻频率）                    │
│ • 患者报告的疗效评估（PGA）                      │
│ • 生活质量快速评估（QoL-5）                      │
│ 工具：手机 app / 智能手表                        │
└─────────────────────────────────────────────────┘
              ↓ 实时数据流
┌─────────────────────────────────────────────────┐
│ 层 2：可穿戴设备（连续或每日）                  │
│ • 心率变异性（HRV）— 炎症指标代理               │
│ • 睡眠质量 — 疾病活跃度的镜像                    │
│ • 步数/活动量 — 功能维持指标                     │
│ 工具：Apple Watch / Oura Ring / 智能腕带       │
└─────────────────────────────────────────────────┘
              ↓ 周度汇总
┌─────────────────────────────────────────────────┐
│ 层 3：周度粪便标志物（每周或每双周）           │
│ • 粪便钙卫蛋白（fCP）— 肠道炎症               │
│ • 粪便乳铁蛋白（fLF）— 肠道损伤                 │
│ • 16S 扩增子测序 — 快速菌群监测               │
│ 工具：邮寄样本盒 / 家庭采样 kit                │
└─────────────────────────────────────────────────┘
              ↓ 月度/季度汇总
┌─────────────────────────────────────────────────┐
│ 层 4：定期深层 Omics（每季度或半年）            │
│ • 宏基因组 — 完整菌群组成                        │
│ • 代谢组 — 菌群功能与宿主代谢                    │
│ • 蛋白质组 — 免疫与屏障标志物                    │
│ • 转录组 — 肠上皮细胞状态                        │
│ 工具：深度测序 + 质谱                            │
└─────────────────────────────────────────────────┘
```

## 数据融合与实时处理

### 处理管道

```
患者数据采集 → 数据质量检查 → 标准化与归一化 
           ↓
       多模态融合引擎
      / | | | \
   PRO HRV 粪便 Omics
    ↓   ↓   ↓   ↓
   → AI 预测模型
       ↓ ↓ ↓ ↓
   → 医生决策支持面板
       ↓
   → 实时提醒 + 干预建议
```

### 关键特征

| 数据源 | 更新频率 | 延迟 | 成本 | 患者负担 |
|--------|---------|------|------|---------|
| PRO (app) | 每日 | 实时 | 低 | 低 |
| 可穿戴 | 连续 | 实时 | 中 | 低 |
| 粪便标志物 | 周/双周 | 1-2 天 | 中 | 中 |
| 深层 Omics | 季度/半年 | 1-2 周 | 高 | 低 |

## 临床应用场景

### 场景 1：MRD 预警（维持期）
- **信号**：粪便钙卫蛋白升高 + HRV 下降 + 周期性睡眠不足
- **AI 判断**：MRD 升高的复发风险评分 80/100
- **医生行动**：启动深层 Omics 检查，考虑强化维持或膳食调整

### 场景 2：药物遵从性监测
- **信号**：患者数周内未上传 PRO、粪便标志物恶化
- **AI 判断**：低遵从性警告 + 建议回访患者
- **医生行动**：主动联系患者，评估遵从性障碍

### 场景 3：诱导缓解的个体化监测
- **信号**：患者 A 第 2 周内镜愈合率高，患者 B 无改善
- **AI 判断**：患者 A 的轨迹向理想方向偏向，B 可能需要升级
- **医生行动**：继续监测 A，提前评估 B 是否需要强化

## 关键实现要素

✅ **患者教育**：清晰解释为何需要持续数据，如何使用反馈  
✅ **用户体验**：简化数据输入，减少患者负担  
✅ **数据安全**：HIPAA 合规、端到端加密、患者数据所有权  
✅ **医生工作流**：集成 EHR，不增加额外工作量  
✅ **持续改进**：患者反馈与真实结果持续优化预测模型  

## 伦理与隐私边界

❌ **不做的事**：
- 强制患者持续监测
- 未经明确同意的数据共享
- 用数据惩罚或优化患者的险金/药物费用

✅ **必须的事**：
- 明确的知情同意与数据所有权
- 定期的数据审计与透明度报告
- 患者的数据访问与删除权利
