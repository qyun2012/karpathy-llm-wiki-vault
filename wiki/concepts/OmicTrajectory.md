# Omic Trajectory（纵向Omic轨迹医学）

## 核心定义

从**横截面诊断（snapshot）**转向**纵向轨迹评估（dynamic trajectory）**的革命性思路：

- **横截面模式**（传统）：某一时刻的单次测量 → 患者分类（缓解/活跃）
- **纵向轨迹模式**（创新）：患者在多维 Omic 空间中的演化曲线 → 预测与干预

## 四维空间中的患者轨迹

每个患者可用 4 维 Omic 向量表示其分子状态：

```
患者 i 在时刻 t：
         ⎡ 转录组特征 ⎤
Omic_t = ⎢ 蛋白质组特征 ⎥
         ⎢ 代谢组特征 ⎥
         ⎣ 微生物组特征 ⎦

轨迹 = {Omic_t0, Omic_t1, Omic_t2, ...} 
      (治疗前 → 早期 → 中期 → 长期)
```

关键问题：患者的轨迹**向着正确方向移动吗**？各维度是否**同步改善**？

## 三类临床轨迹

### 1️⃣ **理想轨迹**（Golden Trajectory）
- **特征**：4 个维度同步向深度缓解方向移动
- **临床表现**：内镜愈合 + 生物标志物正常 + 菌群恢复 + 免疫稳定
- **预后**：长期缓解率高，停药可能性大

### 2️⃣ **部分改善轨迹**（Partial Response Trajectory）
- **特征**：某些维度改善，但维度间改善步调不一致
- **示例**：临床症状缓解，但微生物群仍失调；免疫标志物改善但肠屏障未修复
- **临床意义**：提示需要**维度特异的干预**（如膳食调整针对菌群，或强化营养针对屏障）

### 3️⃣ **偏离轨迹**（Diverging Trajectory）
- **特征**：初期改善后，某维度开始恶化或停滞
- **预警信号**：
  - 微生物群再次失调（MRD 升高）
  - 免疫标志物反弹
  - 粪便钙卫蛋白升高但患者无症状
- **临床决策**：这是**亚临床复发的早期信号**，应启动干预

## 实施算法框架

参见 [[OmicTrajectory-Implementation-Guide]] 获取完整技术细节：
1. **多维 Omic 空间建模**：PCA/t-SNE 降维 + 轨迹拟合
2. **患者特异轨迹预测**：基于患者基线特征与早期反应的预测模型
3. **实时偏离检测**：与预期轨迹比较，识别异常
4. **周期性决策树**：根据检测结果推荐干预

## 与传统模式的关键区别

| 维度 | 传统模式 | 轨迹医学 |
|------|---------|--------|
| **时间视角** | 单时点评估 | 连续纵向评估 |
| **数据维度** | 单标志物或少数指标 | 多维 Omic（转录+蛋白+代谢+菌群） |
| **患者分类** | 离散的"缓解vs活跃" | 连续的个体特异轨迹 |
| **干预触发** | 疾病恶化到临床症状 | **亚临床异常提前识别** |
| **预后预测** | 基线状态 + 早期反应 | **完整的轨迹特征** |

## 临床优势

✅ **早期预警**：在患者感觉症状前 2-4 周识别复发风险  
✅ **维度特异干预**：不同的轨迹偏离对应不同的纠正策略  
✅ **停药可能性评估**：长期深度缓解轨迹的患者复发风险最低  
✅ **个体化监测频率**：轨迹稳定者可减少监测，偏离者增加频率
