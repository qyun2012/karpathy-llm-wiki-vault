# Longitudinal Omic Profile（纵向Omic档案）

## 核心定义

**纵向 Omic 档案**是患者终身的**分子进化记录**，包含：

1. **病程轨迹**：从确诊、诱导缓解、维持缓解到长期随访的完整分子演化
2. **个体特异复发前兆**：该患者特有的生物标志物异常模式（不是人群平均）
3. **治疗响应模式**：对不同治疗方案的个体化反应特征
4. **生涯管理与停药可能性**：支持长期的个体化诊疗决策

## 与其他Omic概念的关系

| 概念 | 时间视角 | 空间视角 | 临床用途 |
|------|---------|---------|--------|
| **OmicPhenotype** | 横截面（单时点） | 多维空间（4维） | 当前患者的分类与诊断 |
| **OmicRemission** | 终点定义 | 多维目标值 | "好"的Omic状态的定义 |
| **OmicTrajectory** | 纵向动态 | 多维轨迹 | 预测与风险预警 |
| **LongitudinalOmicProfile** | 终身纵向 | 个体特异档案 | 长期生涯管理与医疗决策 |

## 档案的四大组成部分

### 1️⃣ **病程轨迹记录**

```
时间轴
│
确诊 → 诱导 → 早期缓解 → 维持 → 长期缓解 → 复发?
│      │      │        │      │
基线   +4w   +12w     +24w   +52w...
Omic  Omic   Omic     Omic   Omic
```

**存储内容**：
- 各时点的完整 Omic 向量（转录+蛋白+代谢+菌群）
- 与治疗方案的对应关系
- 临床结局与生活质量

### 2️⃣ **个体特异复发前兆**

**概念**：不同患者的复发"警报灯"不同

**示例**：
- 患者 A：粪便钙卫蛋白升高是主要警报（对其他患者可能无关紧要）
- 患者 B：特定菌群失调（如 Enterobacter 升高）是关键信号
- 患者 C：免疫标志物（CD4/CD8 下降）是早期预警

**应用**：
- 基于个体档案，医生知道该患者的"正常范围"
- 只需监测该患者的关键指标，无需全面覆盖

### 3️⃣ **治疗响应模式**

**记录**：该患者对过去治疗的响应特征

| 治疗 | 诱导缓解时间 | 缓解深度 | 维持效果 | 副作用 |
|------|-----------|--------|--------|--------|
| IFX | 4-6 周 | 深度 | 良好 (24 月) | 无 |
| AZA | 12 周 | 部分 | 一般 (16 月) | GI 耐受差 |
| VDZ | 8 周 | 中度 | 良好 (28 月) | 无 |

**临床启示**：
- 复发时，优先选择过去效果最好的方案
- 避免过往失败的方案
- 预测新方案的可能效果

### 4️⃣ **停药可能性评估**

**关键指标**：
- **缓解深度与持续时间**：需要 ≥ 1-2 年的完全Omic缓解
- **菌群稳定性**：益菌（Faecalibacterium）连续 ≥ 6 月保持 > 8%
- **免疫标志物恢复**：CD4/CD8 恢复至正常范围，IL-6 < 5 pg/mL
- **复发风险评分**：AI 基于档案数据给出的个体化风险评分 < 20%

**决策框架**：
```
是否可以尝试减量或停药?
     ↓
 ≥2 年完全缓解？
    ↙      ↘
   否       是
   │        ↓
   持续    菌群稳定？
           ↙      ↘
          否       是
          │        ↓
        维持   免疫恢复？
              ↙      ↘
             否       是
             │        ↓
           维持    复发风险 < 20%?
                   ↙      ↘
                  否       是
                  │        ↓
                维持    可尝试减量/停药
                         (强化监测)
```

### 隐私与安全
- ✅ 数据脱敏：去除姓名、日期等直接标识
- ✅ 访问控制：仅患者与其医疗团队可访问
- ✅ 审计日志：所有数据访问与修改均记录
- ✅ 数据加密：传输与存储均采用 AES-256

## 临床应用场景

### 场景 1：复发时的快速决策
- **输入**：新的 Omic 数据显示复发信号
- **查询档案**：过去最有效的 3 个方案
- **输出**：优先选择 IFX（过去 79% 缓解率）+ AZA（协同），预期 6 周缓解
- **时间节约**：减少 3-6 个月的试错时间

### 场景 2：停药可行性评估
- **输入**：患者已 28 个月完全缓解
- **查询档案**：
  - 菌群：Faecalibacterium 12%, 连续 18 月稳定 ✅
  - 免疫：CD4/CD8 = 1.8, IL-6 = 3 pg/mL ✅
  - 复发预测：AI 评分 15/100 ✅
- **输出**："可尝试减量"（保持强化监测）

### 场景 3：新药方案评估
- **输入**：患者要求尝试 JAK 抑制剂
- **查询档案**：
  - 患者过往肠道外炎症（关节炎）有助 JAK 治疗
  - 但过去对 TNF 抑制剂无耐受问题（可优先选择）
- **输出**："建议先坚持 IFX，JAK 作为二线"

## 与其他系统的整合

### 与 EHR 的整合
- 电子病历自动导入患者的临床指标
- Omic 档案与临床记录实时同步

### 与 AI 的整合
- AI 基于档案数据预测：
  - 治疗反应（基于相似患者档案匹配）
  - 复发风险（基于个体特异前兆）
  - 停药可行性（基于完整的缓解深度与持续时间）

### 与患者决策支持的整合
- 患者可查看自己的档案摘要
- 了解自己的"关键指标"与"警报阈值"
- 支持医患的**共享决策**
