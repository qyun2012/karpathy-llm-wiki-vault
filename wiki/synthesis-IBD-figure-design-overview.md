## 📋 **Figure 1：四阶段闭环框架**

### **设计概念**
患者从诊断到停药的完整生命周期，通过四个堆叠圆形 + 螺旋时间线展现四阶段与缓解进程。

### **核心元素**
```
        Phase 1: Diagnosis (Week 0)
        ├─ 采集 4 层基线数据
        └─ 建立个性化参考轨迹 f(t)
                    ↓
        Phase 2: Induction (Week 4/8/12)
        ├─ 三次决策点
        ├─ 轨迹偏离评估
        └─ 分子缓解分类 (CMR/PMR/NMR)
                    ↓
        Phase 3: Maintenance (Month 3-12)
        ├─ MRD 驱动的递进式监测
        ├─ CMR: 3 月检查（复发率 8%）
        ├─ PMR: 1 月检查（复发率 16-20%）
        └─ NMR: 周度检查（复发率 30-35%）
                    ↓
        Phase 4: Long-term (Month 12+)
        ├─ 停药安全门槛评估
        └─ 25-30% 患者成功停药
```

### **配色方案**
- **Phase 1**：蓝色（诊断）
- **Phase 2**：绿→黄→红渐变（诱导期决策）
- **Phase 3**：绿/黄/红（缓解层级）
- **Phase 4**：深紫（长期管理）

### **推荐工具**：Figma / Adobe Illustrator
### **预计工时**：2-3 小时
### **输出格式**：PDF + EPS + Figma 源文件

## 📈 **Figure 2：轨迹医学的数学模型与多维空间**

### **三个 Panel 的组合**

#### **Panel A：Logistic 轨迹曲线**
单个患者的缓解过程参数化：
$$f(t) = \frac{L}{1 + e^{-k(t-t_0)}}$$

- $L$：患者的目标缓解水平
- $k$：个体化的缓解速率（由菌群多样性、基因型等决定）
- $t_0$：预期最快缓解点

展示两个对比患者：
- "快速缓解者"：L=15, k=0.8, t₀=3 周
- "延迟缓解者"：L=15, k=0.4, t₀=6 周 → Week 4 时偏离 1.2σ → 触发干预

#### **Panel B：Mahalanobis 多维空间**
4D 空间中 (CRP, Shannon, Butyrate, CD4/CD8) 的轨迹偏离：
- 灰色云：健康对照的正常值范围
- 绿→黄→红：患者实际轨迹的偏离程度
- 展现"期望轨迹" vs "实际轨迹"的分离

#### **Panel C：维度特异的偏离（雷达图）**
5 维雷达图展现不同维度的贡献：
- 微生物（Microbiota）
- 代谢（Metabolism）
- 屏障（Barrier）
- 免疫（Immune）
- 临床（Clinical）

示例：患者 A 的微生物失调 0.7σ → 触发益生菌干预；患者 B 的免疫活跃 0.8σ → 触发 TDM

### **推荐工具**：R ggplot2 + Plotly（可交互）
### **预计工时**：4-6 小时
### **输出格式**：PDF + 交互式 HTML + PNG


### **9 个干预分支**
| 分支 | 触发条件 | 具体干预 | 监测指标 | 预期效果 |
|-----|--------|--------|--------|--------|
| I1 | σ<0.5，全维度正常 | 继续当前治疗 + 益生菌辅助 | 3 月复查 | 维持缓解 |
| I2 | 免疫活跃 | 升级至 JAK 抑制剂 | 周度 IL-6 | 2-4 周改善 |
| I3-I5 | 微生物失调 | 益生菌/FMT/膳食优化 | 周度 16S | 4 周菌群恢复 |
| I6-I7 | 屏障损伤 | 肠内营养强化 (EEN/MCFA) | 周度 DAO | 2-4 周屏障修复 |
| I8-I9 | 免疫活跃 | 生物制剂升级/频率增加 | 周度 TDM | 3 周谷值达标 |

## 🏗️ **Figure 5：患者即传感器的多源数据融合架构**

### **5 层系统架构**

#### **Layer 1：数据采集（Patient Side）**
- 📱 智能手机 APP：日常 PRO、症状日记、用药依从性
- ⌚ 可穿戴设备：心率、活动度、睡眠质量
- 🚽 智能马桶：排便特征、生物标志物采样
- 🧪 定期采样：周度指尖血、月度粪便、半年成像

#### **Layer 2：安全传输与隐私保护**
- 同态加密（传输加密）
- HIPAA 兼容云平台（AWS HealthLake 或 Microsoft Healthcare）
- GDPR 数据本地化存储
- 端到端加密 (E2EE)

#### **Layer 3：数据融合与处理**
- **标准化**：FHIR 格式转换
- **实时处理**：Kafka/Spark Streaming
  - 缺失值填补
  - 异常值检测
  - 滑窗聚合
  - 特征工程

#### **Layer 4：多模态 AI 融合网络**
```
LSTM Layer: 时间序列特征提取
    ↓ 128-dim 特征
GCN Layer: 特征关系建模
    ↓ 64-dim 图嵌入
Dense Fusion: 多模态融合
    ↓ 192-dim 融合向量
Output: 3 个预测
  ├─ Trajectory Risk Score
  ├─ MRD Loss (分子缓解概率)
  └─ Dimension-specific Deficit
```

#### **Layer 5：临床决策支持输出**
- **患者仪表板**：
  - 交互式轨迹图（自己 vs 期望）
  - 日度进度徽章
  - 红/黄/绿警报系统
  
- **医生仪表板**：
  - 轨迹热力图
  - 维度特异建议
  - 4 周提前风险预测
  - 生物制剂谷值追踪
  
- **EHR 集成**：
  - 自动化 CRP/Cal 医嘱
  - 临床备注触发
  - 药房剂量调整同步

### **关键特征**
- 💾 **联邦学习**：模型在当地医学中心训练，仅共享参数更新
- 🔐 **隐私优先**：患者数据永不离开医学中心
- ⚡ **计算延迟** < 2 小时（从数据上传到临床推荐）
- 🔄 **2-4 周持续改进**（每次参数聚合）

### **推荐工具**：Figma / Draw.io
### **预计工时**：3-4 小时
### **输出格式**：PDF + SVG 矢量图 + Figma 源文件

## ✅ **质量标准检查清单**

### **技术要求**
- [ ] 分辨率：300 DPI（打印清晰）
- [ ] 文件格式：PDF（主）+ EPS（可缩放）+ PNG（网络）
- [ ] 配色：使用 Okabe-Ito 6 色方案（色盲友好）
- [ ] 字体：Helvetica Neue 或 Arial 标准字体
- [ ] 大小：符合 Nature Review 尺寸（85/170 mm）

### **设计要求**
- [ ] 所有图例：清晰独立，可单独理解
- [ ] 颜色对比度：≥ 4.5:1（WCAG AA）
- [ ] 文字可读性：最小字体 9pt
- [ ] 一致性：所有图表使用统一配色与字体

### **科学要求**
- [ ] 数据准确：来自 Kiel 队列（n=150）或已发表文献
- [ ] 结果可重现：附带源代码或脚本
- [ ] 标注完整：标题、轴标签、单位、图例、注释
- [ ] 置信区间：显示统计不确定性（误差棒或阴影）

### **流程检查**
- [ ] 初稿评审：与论文作者过一遍
- [ ] 编辑修改：根据期刊要求调整
- [ ] 最终校对：检查所有文字/数字/符号
